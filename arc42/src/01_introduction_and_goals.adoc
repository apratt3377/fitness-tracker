== Introduction and Goals


fitness-tracker is a project to:

* track my strength workouts
* experiment with microservices architecture
* learn software architecture documentation
* refresher on my java, python, sql skills


=== Requirements Overview

.Contents
The purpose of fitness-tracker is to provide a way for users to keep track of their strength training workouts. They'll also be able to get AI-generated workout plans and notifications about their progress.

This project is also intended to explore different technologies and architectural patterns.

.Main Features
* Users can create, update, and delete workout plans
* Users can view their workout history
* AI-generated analysis of completed workouts
* Simple UI for interacting with the system
* Cloud native microservice architecture

The application should be able to handle multiple users and admins. Users should only be able to see their workouts and recommendations.

The application should be able to store an "unlimited" number of workouts per user.
